## 实验执行摘要

在本次针对 deepseek chat 模型的测试中，一个最反直觉但非常合理的结论是：在面对极度客观、事实性极强的计算机科学基础概念（如 TCP 三次握手）时，即使将 temperature 和 top_p 拉到极值，模型的输出也表现出了极高的稳定性，参数的调节几乎“失效”。

## 核心观察与数据对比

通过横向对比这 15 组不同参数下的回答，可以得出以下具体现象：

- **核心术语绝对锁定**： 无论参数如何变化，SYN（同步序列编号）、SYN-ACK（同步确认）、ACK（确认）这三个核心技术名词的出现顺序和对应关系完全一致，没有任何一次偏离。
- **逻辑拓扑高度一致**： 所有回答的结构均严格遵循“定义 -> 客户端发送请求 -> 服务端确认并请求 -> 客户端最终确认 -> 机制意义”的线性逻辑链条。
- **无“幻觉”或结构崩塌**： 即便在 temperature = 2 且 top_p = 1 这种通常会导致模型胡言乱语或极度发散的极端配置下，模型依然输出了严谨、准确的网络协议解析，没有生造任何不存在的握手步骤。
- **微观措辞的微小扰动**： 参数的改变仅体现在连接词或句式微调上。例如，高温度下偶尔会出现“建立一个可靠的网络连接的过程”与低温度下“建立可靠连接的过程”的区别，但本质信息熵没有变化。

## 深度原理解析：为什么参数“失效”了？

这种现象揭示了大语言模型在处理不同类型知识时的概率分布差异。

在生成文本时，模型会为词表中的每一个候选词计算一个概率。像 TCP 协议这种底层的工程基础知识，在人类语料库中的描述是高度标准化的。这就像在 C 语言或 MSYS2 等底层工具链中配置环境变量一样，路径和依赖关系有着极其严格的规范，没有太多“自由发挥”的空间。

因此，当模型看到“TCP三次握手”这个上下文时，下一个正确词（例如 SYN）的概率分布极其陡峭（概率可能接近 99%）。

- **top_p 的影响被削弱**： 即使 top_p = 1 允许模型考虑所有候选词，但由于正确术语的概率占比极大，那些长尾的非相关词汇依然很难被选中。
- **temperature 的影响被抵抗**： temperature 的作用是平滑概率分布（让高概率变低，低概率变高）。但在极高置信度的事实面前，即使经过 $T=2.0$ 的放大，正确路径的概率依然形成绝对碾压，不足以让模型偏离主干去选择错误的术语。
